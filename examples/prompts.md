# Примеры хороших промптов

## 1. Первичный обзор репозитория

```text
Изучи репозиторий. Найди entrypoint, тесты, команду запуска, ключевые модули и места с наибольшим техническим риском. Сначала только анализ, без изменений.
```

## 2. Узкий багфикс

```text
Воспроизведи падение теста `tests/test_auth.py::test_refresh_token`. Исправь минимальным patch. Не меняй public API и unrelated файлы. После правки прогони только затронутые тесты и объясни root cause.
```

## 3. Review текущих изменений

```text
Сделай review только по текущему diff. Нужны findings по серьезности, с приоритетом на баги, риски регрессии и пробелы в тестах. Краткий summary только после findings.
```

## 4. Безопасный рефакторинг

```text
Сделай рефактор только `src/payments/service.py`. Убери дублирование, не меняй поведение и public API. После правки прогони существующие тесты этого модуля и кратко сравни поведение до и после.
```

## 5. Новый проект

```text
Нужен новый CLI-инструмент на Python для ... Сначала предложи структуру каталогов, зависимости и минимальный MVP. После подтверждения создай проект. В конце дай команды запуска и smoke-test.
```

## 6. Работа с legacy code

```text
Сначала изучи существующие conventions и архитектуру. Потом измени только модуль X под задачу Y. Никаких масштабных переименований и перемещений файлов. После этого прогони релевантные тесты и перечисли остаточные риски.
```

## 7. Генерация документации

```text
Создай developer-facing documentation по текущему repo. Не выдумывай команды и настройки: сначала найди их в конфиге и коде. Все непроверенные вещи помечай как предположение.
```

## 8. Подготовка к PR

```text
Проверь текущий diff как перед PR. Найди возможные баги, поведенческие изменения, пропущенные тесты и сомнительные решения. Затем предложи список точечных улучшений в порядке важности.
```
