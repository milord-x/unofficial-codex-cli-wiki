# Python Bugfix Skill

## Use This Skill When

- есть воспроизводимый Python bug;
- нужен минимальный patch;
- проект использует `pytest` или другой локально проверяемый test runner;
- важно не сломать public API.

## Inputs You Need

- traceback или шаги воспроизведения;
- путь к затронутому модулю, если он известен;
- команда тестов или хотя бы имя failing test.

## Workflow

1. Подтверди симптом и способ воспроизведения.
2. Найди минимальную область кода, связанную с багом.
3. Сформулируй root cause hypothesis.
4. Исправь баг минимальным patch.
5. Если возможно, добавь regression test.
6. Прогони релевантные проверки.
7. Кратко перечисли измененные файлы, проверки и остаточные риски.

## Constraints

- Не делай широкий рефакторинг.
- Не меняй public API без отдельного указания.
- Не трогай unrelated files.
- Не скрывай неопределенность: если причина не подтверждена, скажи это явно.

## Validation

- `pytest path/to/test_file.py`
- `git diff --stat`
- `git diff`

## Output Contract

- root cause;
- что исправлено;
- какие тесты прогнаны;
- какой риск регрессии остался.
