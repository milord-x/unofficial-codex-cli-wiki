# Repository Documentation Skill

## Use This Skill When

- нужно создать или обновить техническую документацию по существующему репозиторию;
- важна точность команд, путей и зависимостей;
- нельзя выдумывать детали, которых нет в коде.

## Workflow

1. Найди README, build/test configs и entrypoints.
2. Извлеки реальные команды запуска, тестов и сборки.
3. Сопоставь структуру каталогов с фактическим кодом.
4. Сгенерируй документацию только по подтвержденным фактам.
5. Все непроверенные детали пометь как предположение.
6. Проверь итоговый diff на выдуманные команды и пути.

## Constraints

- Не придумывай setup steps, если их нет в repo.
- Не копируй целиком большие куски кода в документацию.
- Не подменяй проектные conventions своими.

## Validation

- проверить команды по `package.json`, `pyproject.toml`, `Makefile`, `README`;
- проверить пути файлов через поиск по репозиторию;
- сравнить документацию с текущим `git diff`.

## Output Contract

- краткая карта проекта;
- подтвержденные команды;
- ограничения и предположения;
- список файлов документации, которые изменены.
